PlantBid – Funkční a technická specifikace

Zápočtová práce - Specifikace

1. Úvod

PlantBid je jednoduchá webová aukční aplikace zaměřená na prodej a nákup rostlin mezi uživateli. Cílem aplikace je umožnit:

  • vytvářet aukce,
  • procházet aktivní aukce,
  • přihazovat,
  • sledovat historii příhozů.

Tento dokument obsahuje funkční a technickou specifikaci pro účely předmětu Tvorba webových aplikací. Projekt vychází z původní databázové aplikace a je rozšířen tak, aby splnil požadavky zadání.

2. Funkční specifikace

2.1 Datový konceptuální model

Aplikace pracuje s několika základními entitami a jejich vztahy.

Entity

  • Uživatel – přihlašovací údaje, role, základní informace.
  • Aukce – prodávaná rostlina, ceny, čas ukončení, obrázek.
  • Příhoz – nabídka uživatele v aukci.
  • Kategorie (volitelné) – tematické zařazení aukcí.

Vztahy

  • Uživatel 1:N Aukce
  • Uživatel 1:N Příhoz
  • Aukce 1:N Příhoz
  • Kategorie 1:N Aukce (volitelné)

2.2 Charakteristika funkčností

Základní chování aplikace z pohledu uživatele.

Hlavní funkce

  1. Registrace a přihlášení – vytvoření účtu, login, logout.
  2. Správa profilu – základní údaje, přehled vlastních aukcí a příhozů.
  3. Procházení a filtrování aukcí – seznam aktivních aukcí, filtrování, vyhledávání.
  4. Zobrazení detailu aukce – podrobnosti, historie příhozů, graf.
  5. Přihazování – validace výše příhozu, aktualizace aktuální ceny.
  6. Zakládání a správa vlastních aukcí – vytvoření, úprava, případně předčasné ukončení.
  7. Administrace – blokace uživatelů, mazání problémových aukcí.

2.4 Uživatelské grafické rozhraní

Přehled hlavních obrazovek aplikace.

  • Domovská stránka – seznam aktivních aukcí, základní filtry, vyhledávání.
  • Detail aukce – informace o rostlině, historie příhozů, graf vývoje ceny (Canvas).
  • Přihlášení / registrace – formuláře s validací.
  • Profil uživatele – vlastní aukce, příhozy, oblíbené aukce (uložené v localStorage).
  • Administrace – seznam uživatelů a aukcí, základní moderace.

3. Technická specifikace

3.1 Datový logický model

Tabulka users

  • id (PK)
  • username
  • email
  • password
  • role (user/admin)
  • created_at

Tabulka auctions

  • id (PK)
  • user_id (FK)
  • title
  • description
  • image
  • starting_price
  • current_price
  • category_id (FK, volitelné)
  • status (active/ended/cancelled)
  • end_time
  • created_at

Tabulka bids

  • id (PK)
  • auction_id (FK)
  • user_id (FK)
  • bid_amount
  • bid_time

Tabulka categories (volitelné)

  • id (PK)
  • name
  • slug

3.2 Architektura

Aplikace používá klasické rozdělení klient – server.

Klient

  • HTML5 (šablony, formuláře)
  • CSS3 (vzhled, responzivita)
  • JavaScript (ES6):
    • fetch API pro načítání JSON dat,
    • odpočet času do konce aukce,
    • filtrace a řazení aukcí,
    • localStorage pro oblíbené aukce,
    • HTML5 Canvas pro graf historie příhozů.

Server

  • PHP 8+ s OOP architekturou,
  • repozitáře pro práci s databází,
  • REST-like endpointy /api/*,
  • PDO pro komunikaci s MySQL.

Databáze

  • MySQL s tabulkami users, auctions, bids, případně categories.

3.3 Popis tříd

Database

  • Správa PDO připojení k databázi.

User

  • Atributy: id, username, email, passwordHash, role.
  • Metody: gettery, isAdmin().

Auction

  • Atributy: id, owner, title, description, ceny, časy, status.
  • Metody: isActive(), getTimeRemaining(), raisePrice().

Bid

  • Atributy: id, auction, user, amount, time.

Repozitáře

  • UserRepository – práce s uživateli.
  • AuctionRepository – získávání a ukládání aukcí.
  • BidRepository – příhozy a jejich historie.

Servisy

  • AuthService – login, logout, správa session.
  • BidService – validace a provedení příhozů.
